Note: every interactive element from the previous round has been exercised end-to-end (Playwright, not eyeballed) and confirmed working — command palette (nav actions, filtering, keyboard selection), the row-actions dropdown (Open/Rename/Share/Delete), bulk bar Export, pagination prev/next, table sort (which now actually reorders rows, not just flips the arrow), the New Case form, and the drawer's Download/Share. Fixed along the way: two functional gaps where controls looked interactive but had no handler at all, plus two runtime bugs (an inline-edit double-blur crash, and dark-mode table text stuck at the light-mode color due to a browser table-inheritance quirk).

New this round: Typography scale, a paired dark color-swatch grid, an icon set, alert/banner variants, breadcrumbs, avatars & presence, a stepper, a multi-select combobox, chat bubbles with a simulated assistant reply, a drag-and-drop Kanban board, a redline/diff view, unread nav badges, and an explicit keyboard-focus demo.

Just added: data-table rows now open the same detail drawer as documents, populated with real vehicle data (plate, model, APK, WAM); a header notification bell with a live unread count; a Deadlines & Reminders section with a ticking live countdown to the next deadline, check-off-to-complete rows, and an add-reminder form with a repeat cadence picker; and an Appointments section — a real month calendar with an agenda pane, where each event with a location gets an "Open in Maps" link and every event can be exported as a downloadable .ics file. All wired up the same way as the rest (click-tested locally, not yet run through the Playwright pass). Still deliberately left as backlog: domain-specific components (real plate input, case-status pipeline, styled metadata display), mobile-specific patterns, and a WCAG contrast audit. Tell me if it's ready to lock in as the spec.
